Introduction

深度學習飛快進步，主要原因不是強大的硬體、資料集、模型，而是新的想法、 演算法、架構，如ILSVRC14資料量沒變，然而參數卻少於AlexNet的12倍， 並獲得勝利，這是利用新的架構所得到的成果，就好比R-CNN演算法。

值得注意的是為了加速行動與遷移運算發展，演算法變得更為重要， 因此提出了新的架構，而不是單單的為了成功率，在設計過程中，保持著1.5b的加乘法限制， 所以不僅是在學術探討，而是能運用在現實生活中，甚至運用在大型。

Motivation and high level considerations

增加神經元與網路層是最簡單提升模型的方法，但這有兩個缺點。

大模型意味著多參數，容易讓模型overfitting，尤其是在資料不足的情況下，而且高質量的資料並不容易取得。

另一缺點是，會增加電腦運算量，兩層卷積，運算量會以指數增加，如果大多數權重都趨近於零，則會浪費大量的運算。

解決這兩個問題可以將全連接層替換成稀疏連接層，甚至卷積層也增加稀疏性。理論基礎建立在 “Provable bounds for learning some deep representations”上。





然而，電腦對非均勻稀疏性結構運算很差，甚至將運算次數降低100倍，在lookup和caches misses佔非常大的計算資源。 而隨著演算法改進、數據庫的優化、CPU與GPU的升級，這種快速密集矩陣乘法，只會不斷擴大。目前大多模型， 都是利用卷積結構來獲得空間稀疏性，但本質上還不算還是稀疏連結，因為卷積核還是得逐點計算。 傳統的卷積網路在特徵維度上使用隨機、稀疏的連接，打破對稱性並提高效率，後來使用全連接層，目的是能平行計算且更好優化。

既然無法得到完整的稀疏結構，那就在filter level上實踐一個具有稀疏性的架構， 根據“ON TWO-DIMENSIONAL SPARSE MATRIX PARTITIONING: MODELS, METHODS, AND A RECIPE”這篇paper表明， 將稀疏矩陣聚集為相對密集的子矩陣，能保持稀疏性，也能得到較好的計算能力。



根據上述問題，paper提出了Inception架構，內容包括了上述的稀疏結構。

儘管Inception架構在結果上的到好的成果，不過是不是真的因為這個架構，還得驗證。



Architectural details

Inception架構目的是要找到一個“局部最佳稀疏結構”近似於“稠密結構”， 並且需要具有translation invariance意味著需要使用卷積結構建構，並將此結構重複使用。 底層網路的作用是展現原圖的局部區域，因此能在inception結構中使用多個卷積核， 然後使用一層1x1的卷積進行聚合。為了避免patch對齊問題，inception結構的卷積核限制在1x1、3x3、5x5， 並加入一個池化層，平行於卷積層，有利於結果產生。如下圖。



Inception結構網上堆疊，特徵相關性會越來越低，他們間的聚集程度也會縮小， 則3x3、5x5的卷積核應該增多，以捕捉特徵。

Naive有個問題，雖然5x5卷積核不多，不過運算代價相當高，加入池化層更加嚴重， 結合後的filter的會非常大。

因此提出新架構，利用降維，減少運算量。而low dimensional embedding能讓輸入降維的情況下， 依舊保留輸入資訊。不過像這種壓縮過的資料，並非稀疏結構，因此使用1x1卷積在3x3、5x5之前，max pooling之後， 進行降維，並使用ReLU當作1x1卷積的線性激活。

GoogLeNet大多都是由inception結構所組成，由時候會將stride==2，將輸出filter將為原本的1/2。 從技術角度來說，在底層網路使用傳統的卷積架構，高層的網路使用inception架構，這能夠高效能使用記憶體來訓練。

Inception結構的組成，能增加神經元數量，卻不會明顯增加運算量。主要是因為使用了1x1的卷積進行降維， 並使用不同維度的卷積核，去取得不同尺度的特徵。

計算參數的改進，允許架構增加寬度，及深度，並不增加運算難度。 這版本的inception架構訓練後結果會比naive版本差一些，不過可以降低計算成本。





GoogLeNet

作者們使用GoogLeNet當作隊伍名稱參加ILSVRC14，並使用相同名字命名用inception建構後的網路， 主要是致敬Yann LeCuns的LeNet-5。GoogLeNet使用經過降維的inception架構建構，並提升了結果。



輸入為RGB 3x224x224的圖像，inception架構中，在3x3、5x5之前，max pooling之後加入1x1的卷積， 並使用ReLU當作線性激活。

該網路考慮了效能與實用性，能應用在運算資源較少的設備。整個網路包含參數的僅有22層。 在輸出前使用average pooling，這主要是參考ＮＮ架構，並在average pooling後加入dense layer， 根據label sets做調整。則成功率提升0.6%。

GoogLeNet由於深度過深，要如何進行有效的反向傳播是個問題。作者發現，在網路中間層的特徵， 也有一定的鑑別度。透過中間層的輸出，增加反向傳播時的訊號，並將其的正規化。這些輔助分類器， 使用卷積架構進行，並將這些loss值乘0.3加入至總loss值。正式輸出時則不使用輔助分類器。

結構如下：
  • Average pooling layer 大小為5x5，strides為3，4a輸出為4x4x512，4b輸出為4x4x528。
  • 具有128個filter的1x1的卷積，使用ReLU當作線性激活，並用於降維。
  • 具有1024個輸出的dense層，並使用ReLU當作線性激活，並設定dropout為0.7。






Training methodology

使用DistBelief對神經網路進行訓練，並使用model and data parallelism。儘管作者使用CPU進行訓練， 但GPU訓練速度還是優於前者，主要限制還是記憶體大小。訓練時的optimizer使用的是SGD，並將momentum調為0.9， 固定的learning rate。使用“Acceleration of stochastic approximation by averaging.” 的Polyak averaging來決定最後模型。

關於資料抽樣的部分，作者針對比賽更改多次抽樣方式，也修改多次超參數，因此很難給出有效的訓練參數。 更麻煩的事，有些模型是利用小尺度的圖片進行訓練，有些是大尺度的圖片。不過透過某些規則，能有較好的表現。 此外根據“Some Improvements on Deep Convolutional Neural Network Based Image Classification” 提出的photometric distortions，能有效防止overfitting。使用random interpolation methods， 來比較晚的調整圖片大小，並結合超參數變化，因此無法確定影響結果的實際原因為何。



ILSVRC 2014 classification challenge setup and result

ILSVRC detection task是一個產生在200個類別中，對物件產生bounding box的任務。 如果圖片的類別分類正確，且與bounding box重疊的部分小於50%，則設為正確，否則設為錯誤。 classification task中，一張圖片可能含有許多問題，造成計算結果方式複雜， 因此使用mean average precision (mAP)來計算成功率。

GoogLeNet的偵測方式類似於R-CNN的方式，不同之處在於，前者包括inception架構。 此外，結合Selective Search approach與multi-box，增加了模型的招回率。為了降低false positives， superpixel大小會增加為兩倍，這將selective search algorithm的區域生成減少一半。 作者們總共新增了200個region proposals，其中60%來自R-CNN，覆蓋率從92%增加至93％。減少proposals數量， 反而增加覆蓋率的mean average precision 1%。使用inception架構，成功率從40%提升至43.9%。與R-CNN不同， GoogLeNet並沒有使用bounding box regression。



GoogLeNet的最好結果，相比2013年的mean average precision幾乎快翻一倍。最好的幾個模型， 都是使用convolution network。external data來自ILSVRC2012，主要是用來pre-train，並對模型進行細化。 有些隊伍是使用localization data，而GoogLeNet則沒有使用localization data進行預訓練。



在signal model中，Deep Insight反而優於GoogLeNet，在mean average precision僅減少0.3個百分點， 而GoogLeNet明顯在ensemble中效果較好。



Conclusions

實驗結果證明，使用稠密矩陣近似於稀疏矩陣，能對深度學習在電腦視覺領域進行改善。 Inception架構主要優點是，增加適度的電腦資源，就能對結果進行顯著的改善。 儘管沒有context也沒有bounding box regression，但依舊持有競爭力，這證明了inception架構的強大。 其實使用更深、更廣的神經網路，依然可以達到此成功率，但是也更消耗電腦資源，而inception架構也證明了， 使用稀疏結構能增加效率。在未來工作，將在Arora的基礎上，設計自動化創建更稀疏、更精密的結構，讓inception模型， 能在其他領域使用。



Ｑ：inception中的convolution是如何結合？

Ａ：inception在進行四種運算時，最後的輸入圖片大小，必須與輸出圖片大小相等，而channel部分，則是將四種運算結果串接在一起。



Ｑ：1x1卷積，是如何進行降維？

Ａ：1x1卷積，在對一張channel等於1的圖，就好比對整張圖乘以一個常數。如果channel大於1時， 類似對圖片中的每個點進行fully connection layer的運算。channel的會隨著1x1卷積的filter進行改變，以達到減少運算量。



Ｑ：為何GoogLeNet不使用一般的反向傳播？要使用輔助分類器進行參數更新。

Ａ：一般來說，像這種深層網路容易會造成梯度消失及爆炸， 會造成上述兩種原因還是因為BP是基於chain rule，如果相乘的值都大於1，則梯度爆炸，反之則梯度消失。

Ｑ：為什麼inception架構需要translation invariance？

Ａ：translation invariance翻譯為“平移不變”。我認為有兩個原因：
  • inception架構類似於convolution一樣，只是一個擷取圖片特徵的網路層，所以使用convolution為基礎建置。
  • Convolution在做影像辨識時，如果原影像進行向左偏移(datasets中沒有向左偏移的資料集)，神經網路依然可以判斷出來。


Ｑ：機器學習中，稀疏矩陣的重要性。

Ａ：含有稀疏性的矩陣能大幅降低運算量，並且在實際大型網路中發現，大多矩陣都具有稀疏性。而稀疏矩陣會產生兩大問題：
  • 儲存空間的浪費→稀疏矩陣中，含有大多不具有訊息的數值。
  • 計算資源的浪費→雖然稀疏矩陣能降低運算量，不過大多運算都是無意義的，從另一方面來看，反而是浪費計算資源。


Ｑ：深度學習產生稀疏矩陣原因

Ａ：產生稀疏矩陣其中一種原因是L1norm，其數學推導在此。
